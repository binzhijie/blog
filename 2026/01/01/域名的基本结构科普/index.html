域名的基本结构科普

最后更新: 1月 01, 2026 作者: ZhijieBin 字数: 736 预计阅读时间: 2min

一、域名的基本结构

域名采用从右到左逐级细分的层次结构，每一级用点（.）分隔。例如：

从右往左看，层级如下：

  • .com —— 顶级域名（TLD）
  • example —— 二级域名
  • blog —— 三级域名（子域名）
  • www —— 四级域名（更细的子域名）

注意：在日常交流中，“一级域名”通常指的是“完整注册的主域名”，比如 example.com，而不是仅 .com。这容易造成术语混淆，所以更推荐使用“顶级域名、二级域名”等标准术语。

二、各层级的定义

1. 顶级域名（Top-Level Domain, TLD）

  • 位于最右边，是域名系统的最高层级。
  • 分为两类：
    • 通用顶级域（gTLD）：如 .com、.net、.org、.xyz 等；
    • 国家代码顶级域（ccTLD）：如 .cn（中国）、.uk（英国）、.jp（日本）等。
  • 由 ICANN（互联网名称与数字地址分配机构）统一管理。

例如：在 example.com 中，.com 就是顶级域名 。

2. 二级域名（Second-Level Domain, SLD）

  • 位于顶级域名左侧，是用户实际注册的部分。
  • 构成你网站的“主名称”，比如：
    • baidu.com 中的 baidu
    • taobao.cn 中的 taobao

通常所说的“一级域名”在实践中就是指“二级域名 + 顶级域名”的组合，例如 baidu.com 被称为一个“一级域名” 。

3. 三级域名（Third-Level Domain）

  • 也叫子域名（Subdomain），是用户在主域名下自行创建的。
  • 用于划分不同服务或功能模块，例如：
    • mail.google.com → mail 是三级域名，表示邮箱服务；
    • shop.taobao.com → shop 是淘宝的店铺子域；
    • www.example.com → www 是最常见的三级域名（表示万维网）。

子域名可以无限嵌套，因此存在四级、五级……域名 。

4. 四级、五级……域名？

  • 技术上，DNS 支持最多 127 级，但实际使用中极少超过 4~5 级。

  • 例如：

    • .com → 顶级
    • example → 二级
    • service → 三级
    • user → 四级
    • api → 五级
    • dev → 六级

这种多级结构多用于大型企业或微服务架构中，用于隔离不同环境或模块 。

三、“一级域名”到底指什么？术语澄清

这里存在两种理解方式：

说法 含义 是否推荐
严格 DNS 层级定义 一级域名 = 顶级域名（如 .com） ✅ 准确但少用
日常/建站行业习惯 一级域名 = 用户注册的完整域名（如 example.com） ✅ 更常用

国内很多资料将 example.com 称为“一级域名”，而 www.example.com 为“二级域名” 。这种说法虽不完全符合 DNS 标准，但在实际使用中被广泛接受。

四、总结图示

  • 你花钱注册的是“二级域名 + 顶级域名”（如 yourname.com）。
  • 子域名（三级及以上）可免费自行创建，无需额外注册。
